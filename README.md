#### 软件架构
- MASM.EXE:

    汇编器。MASM是Microsoft Macro Assembler的缩写，用于将人类可读的汇编语言源代码（如.asm文件）转换为目标机器代码（.obj文件）。这个过程中，汇编器会翻译指令、分配内存位置、处理符号和表达式等，为后续的链接步骤准备代码。

- LINK.EXE:
    
    链接器程序。在汇编语言编程中，源代码首先通过MASM.EXE（汇编器）转换成目标代码（OBJ文件）。LINK.EXE负责将这些目标代码文件、库文件以及其他必要的组件链接在一起，生成最终可执行的EXE文件。它处理程序的段定义、外部引用解析、重定位等工作，确保程序可以在操作系统上正确运行。

- debug.exe:

    调试工具。Debug是一个古老的DOS时代的调试工具，用于检查和修改内存内容、执行汇编语言指令、跟踪程序执行等。在开发8086汇编语言程序时，开发人员可以使用debug.exe来调试他们的程序，比如查看内存状态、设置断点、单步执行等，以找出并修复程序中的错误。

- snake.asm:

    源代码文件。这是贪吃蛇游戏的主程序源代码，用汇编语言编写。.asm文件包含了构成游戏逻辑的所有指令、数据定义、子程序等。开发者在这个文件中编写控制蛇的移动、食物生成、碰撞检测、得分计算等游戏逻辑。

- SNAKE.OBJ:

    目标代码文件。当MASM.EXE处理完snake.asm后，生成的就是这个.obj文件。它是汇编语言源代码经过编译后的二进制形式，但还不能直接作为可执行文件运行，因为它可能包含未解决的外部引用或需要进一步链接处理的段信息。

- SNAKE.EXE:

    可执行文件。这是最终的产品，由LINK.EXE将SNAKE.OBJ（以及项目中可能引用的任何其他对象文件或库）链接生成的。这是一个可以直接在支持8086处理器的DOS环境下运行的游戏程序。用户通过双击或在命令行输入文件名即可启动游戏。

#### 操作介绍

基本操作介绍：
- W：向上移动
- A：向左移动
- S：向下移动
- D：向右移动
- P：暂停游戏
- R：重新开始
- ESC：退出游戏
- Num1：最慢的速度
- Num2：速度二档
- Num3：速度三档
- Num4：速度四档
- Num5：最快速度
- space：从开始界面跳转游戏开始
